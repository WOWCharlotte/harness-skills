# 测试团队提示词模板

## 团队架构

| 角色 | 职责 | 输入 | 输出 |
|------|------|------|------|
| 需求分析专家 | 需求澄清与拆分 | 原始需求文档 | 需求清单 |
| 高级测试工程师 | 测试点提取与策略 | 需求清单 | 测试点清单 |
| 测试专员 | 测试用例编写 | 测试点清单 | 测试用例 |

## 目录结构

```
test-team/
├── README.md                      # 本文件
├── specs/                         # 角色规范
│   ├── requirements-analyst.md     # 需求分析专家
│   ├── senior-test-engineer.md     # 高级测试工程师
│   └── test-specialist.md          # 测试专员
├── templates/                      # 协作模板
│   ├── requirement-handoff.md      # 需求交接模板
│   └── test-point-handoff.md       # 测试点交接模板
└── examples/                       # 使用示例
    └── demo/                        # 演示案例
        └── README.md                # 登录功能完整示例
```

## 协作流程

```
原始需求
    │
    ▼
┌─────────────────────┐
│  需求分析专家        │
│  Requirements Analyst │
└─────────────────────┘
    │
    ▼ (需求清单 JSON)
┌─────────────────────┐
│  高级测试工程师      │
│  Senior Test Engineer│
└─────────────────────┘
    │
    ▼ (测试点清单 JSON)
┌─────────────────────┐
│  测试专员            │
│  Test Specialist    │
└─────────────────────┘
    │
    ▼
测试用例库
```

## 使用说明

各角色的提示词规范见 `specs/` 目录。
交接标准格式见 `templates/` 目录。
