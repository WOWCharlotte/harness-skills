# 测试团队使用示例

## 场景

以"用户登录功能"为例，展示三角色协作流程。

## 原始需求

> **用户登录功能需求**
>
> 用户需要通过用户名和密码登录系统。系统应验证用户凭据，正确时允许访问，错误时拒绝并提示。安全起见，连续失败5次后应锁定账户10分钟。

---

## Step 1: 需求分析专家产出

**输入**: 原始需求文档
**输出**: 需求清单 (requirement-handoff.json)

```json
{
  "handoff_id": "HO-REQ-001",
  "requirements": [
    {
      "id": "REQ-001",
      "title": "用户登录功能",
      "description": "用户可以使用用户名密码登录系统",
      "acceptance_criteria": [
        "用户名密码正确时登录成功",
        "用户名密码错误时提示错误信息",
        "连续失败5次后账户锁定10分钟"
      ],
      "dependencies": [],
      "priority": "high",
      "notes": ""
    }
  ],
  "ambiguous_points": [
    "密码强度要求未明确",
    "是否支持第三方登录未说明"
  ],
  "assumptions": [
    "默认使用本地数据库认证",
    "会话超时时间为30分钟"
  ],
  "excluded_scope": [
    "第三方登录（微信、GitHub等）"
  ]
}
```

## Step 3: 测试专员产出

**输入**: 测试点清单
**输出**: 测试用例

### TC-001: 用户名密码正确时登录成功

| 字段 | 内容 |
|------|------|
| 需求ID | REQ-001 |
| 测试点ID | TP-001 |
| 优先级 | P0 |
| 执行方式 | 手工 |

**前置条件**: 用户已注册，账户状态正常

**测试数据**:

| 变量 | 值 |
|------|-----|
| username | test_user_001 |
| password | Test@123456 |

**测试步骤**:

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | 打开登录页面 | 显示登录表单 |
| 2 | 输入用户名 test_user_001 | 用户名显示在输入框 |
| 3 | 输入密码 Test@123456 | 密码显示为 *** |
| 4 | 点击登录按钮 | 登录成功，跳转首页 |
| 5 | 检查页面显示用户信息 | 显示用户昵称或欢迎信息 |

**后置条件**: 退出登录，清除会话

---

### TC-002: 密码错误时登录失败提示

| 字段 | 内容 |
|------|------|
| 需求ID | REQ-001 |
| 测试点ID | TP-002 |
| 优先级 | P0 |
| 执行方式 | 手工 |

**前置条件**: 用户已注册

**测试数据**:

| 变量 | 值 |
|------|-----|
| username | test_user_001 |
| password | WrongPassword123 |

**测试步骤**:

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | 打开登录页面 | 显示登录表单 |
| 2 | 输入用户名 test_user_001 | 用户名显示 |
| 3 | 输入密码 WrongPassword123 | 密码显示为 *** |
| 4 | 点击登录按钮 | 提示"用户名或密码错误" |
| 5 | 检查页面未跳转 | 仍停留在登录页 |

---

### TC-003: 连续失败5次后账户锁定

| 字段 | 内容 |
|------|------|
| 需求ID | REQ-001 |
| 测试点ID | TP-003 |
| 优先级 | P1 |
| 执行方式 | 手工 |

**前置条件**: 测试账户当前失败次数为4

**测试数据**:

| 变量 | 值 |
|------|-----|
| username | lock_test_user |
| password | 任意错误密码 |

**测试步骤**:

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | 输入用户名 lock_test_user | 用户名显示 |
| 2 | 输入错误密码 | 密码显示为 *** |
| 3 | 点击登录按钮 | 提示错误（失败第5次） |
| 4 | 等待1秒 | - |
| 5 | 再次输入正确密码尝试登录 | 提示"账户已被锁定" |
| 6 | 记录锁定提示信息 | 显示锁定剩余时间 |

**后置条件**: 等待10分钟后验证账户自动解锁
